# ТЗ: Система кастомной цветовой индикации для сущностей «Репетитор» / «Группа»

## 1. Контекст и цель

**Проблема:** При большом количестве записей (занятия, ДЗ, финансы) от разных репетиторов или групп они визуально сливаются — пользователю сложно быстро отличить, к кому относится элемент.

**Решение:** Пользователь назначает уникальный цвет (цветовую метку) для репетитора или группы; этот цвет применяется как визуальный индикатор ко всем связанным сущностям (уроки, ДЗ, при необходимости — финансы).

**Критические ограничения:**
- Цветовая индикация **не должна ломать читаемость:** не заливать весь фон ярким цветом — использовать мягкую подсветку (background opacity 10–15%), цветную левую рамку (`border-left`) или цветные бейджи/иконки.
- Цвета должны корректно смотреться в **светлой и тёмной теме** (безопасные палитры или CSS-переменные).
- **Не использовать внешние JS-библиотеки** для color-picker: нативный `<input type="color">` или предзаданная палитра из 6–8 цветов.

## 2. Юзер-кейсы и решения (Было → Станет)

### UC-01: Выбор цвета

| Аспект | Описание |
|--------|----------|
| **Было** | Нет возможности назначить цвет репетитору/группе. |
| **Станет** | Пользователь назначает цвет в месте управления связью с репетитором или группой. |
| **Где** | **Ученик:** в карточке репетитора (`tutor_card`) или в списке «Мои репетиторы» (`my_tutors`) — кнопка/ссылка «Настроить цвет» открывает модалку или отдельный блок с выбором цвета для этого репетитора. Для группы — в карточке группы или в фильтре расписания (если есть список «Мои группы») аналогичный выбор цвета для группы. **Репетитор:** в «Мои ученики» (`my_students`) — для каждой своей группы возможность задать цвет (в карточке группы / в блоке редактирования группы). Цвет репетитора для «себя» не требуется (репетитор видит только свои сущности). |
| **Как** | Форма выбора: либо предзаданная палитра из 6–8 безопасных цветов (кнопки-квадраты), либо нативный `<input type="color">` с подсказкой «или свой цвет». Сохранение — по кнопке «Сохранить»; при необходимости AJAX без перезагрузки страницы. |

### UC-02: Отображение в Расписании

| Аспект | Описание |
|--------|----------|
| **Было** | Карточки/строки уроков выглядят одинаково независимо от репетитора/группы. |
| **Станет** | **Десктоп:** в таблице расписания (`core/lessons_rows.html`) каждая строка урока получает визуальный индикатор выбранного цвета (например, левая рамка 4px и/или лёгкий фон с opacity 10–15%). Цвет берётся от репетитора урока (для ученика) или от группы/ученика (для репетитора: цвет группы, если урок групповой). **Мобильная версия:** карточка урока (`mobile/index.html` — `.lesson-card`) — та же логика: левая рамка и/или мягкий фон. |
| **Ограничение** | Не заливать всю карточку ярким сплошным цветом — только подсветка/рамка, текст остаётся в контрастных цветах темы. |

### UC-03: Отображение в Домашних заданиях

| Аспект | Описание |
|--------|----------|
| **Было** | Список ДЗ (десктоп `my_assignments`, мобильный `my_assignments`) не выделяет задания по репетитору. |
| **Станет** | Каждая карточка/строка ДЗ визуально помечена цветом репетитора, выбранным учеником (тот же приём: левая рамка и/или фон с низкой прозрачностью). На странице ученика цвет однозначно привязан к репетитору, задавшему ДЗ. |
| **Ограничение** | Сохранять читаемость текста и статусных бейджей (pending, submitted и т.д.). |

### UC-04: Удаление / сброс цвета

| Аспект | Описание |
|--------|----------|
| **Было** | Нет функции «цвет». |
| **Станет** | В той же форме, где задаётся цвет (модалка/блок настроек репетитора или группы), предусмотрена кнопка «Убрать цвет» или «Сбросить». После сброса сущность отображается без кастомного цвета (стандартное оформление — как сейчас). В БД поле цвета хранится как NULL или пустая строка. |

## 3. Требования к доступности (контрастность)

- **Текст на фоне:** При применении цвета только как подсветка (рамка + фон с opacity 10–15%) основной текст карточки/строки остаётся на стандартном фоне темы (`--tg-card-bg`, `--bg-white` и т.д.). То есть цвет **не используется как фон для текста** — контрастность не страдает.
- **Цветная рамка/бейдж:** Рамка и мелкие индикаторы (точка, полоска) — декоративные; не единственный способ передачи информации. Назначение урока/ДЗ по-прежнему видно из текста (имя репетитора, группы, предмет).
- **Палитра выбора:** Если используется предзаданная палитра — подобрать 6–8 цветов, достаточно различимых друг от друга и не конфликтующих с тёмной темой (не слишком тёмные на тёмном фоне). Рекомендуется проверка комбинации «индикатор + фон» в обеих темах.
- **Минимальные требования:** Соответствие WCAG 2.1 Level AA для основного текста (контраст не ухудшается за счёт введения цветовой метки).

## 4. Критерии приёмки

1. **Выбор и сохранение:** Пользователь может назначить цвет репетитору (в контексте связи ученик–репетитор) и группе (ученик — для своей группы; репетитор — для своей группы). Цвет сохраняется в БД и отображается при следующих загрузках.
2. **Расписание:** На главной (десктоп и мобильная версия) каждая карточка/строка урока визуально помечена выбранным цветом (рамка и/или мягкий фон), без ущерба читаемости.
3. **Домашние задания:** В списках «Мои задания» (десктоп и мобильная версия) каждая запись ДЗ помечена цветом репетитора.
4. **Сброс:** Пользователь может убрать цвет; после сброса отображение без кастомного цвета.
5. **Темы:** Один и тот же сохранённый цвет корректно отображается в светлой и тёмной теме (без «исчезающих» или нечитаемых индикаторов).
6. **Без тяжёлых библиотек:** Используется нативный `<input type="color">` и/или предзаданная палитра; без подключения сторонних color-picker.
7. **Совместимость:** Нет конфликтов с текущими шаблонами (`core/templates/core/`, `core/templates/mobile/`), стилями (`static/css/style.css`) и CSS-переменными темы (`--tg-*`, `--primary`). Существующая функциональность (фильтры, пагинация, модалки) не ломается.
8. **Доступность:** Контраст основного текста сохраняется; цветовая метка не является единственным способом идентификации сущности.

---

## 5. Стек и границы

- **Стек:** Django, нативный HTML/CSS, Vanilla JS.
- **Где не менять логику:** Авторизация, роли, права доступа к урокам/ДЗ остаются без изменений; добавляется только хранение и отображение цвета.
- **Финансы:** В рамках данного ТЗ приоритет — расписание и ДЗ. Отображение цвета в разделе «Финансы» допускается как следующий шаг (та же модель данных и приём с CSS-переменной), но в минимальный объём реализации может не входить.
