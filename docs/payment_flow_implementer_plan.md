# План для Implementer: Синхронизация флоу оплаты (Desktop → Mobile)

## Контекст
- Проект: **Django** (не PHP). Бэкенд общий для desktop и mobile.
- Эндпоинты уже есть: `payment_receipts` (страница), `submit_receipt` (POST), `approve_receipt` (POST), `reject_receipt` (POST).
- Мобильные страницы `mobile/payment_receipts_student.html` и `mobile/payment_receipts_tutor.html` уже существуют и используют те же URL/формы. Нужно довести навигацию, убрать legacy и добавить просмотр чека в модалке.

## Задачи (по приоритету)

### 1. Навигация — core/templates/mobile/base.html
- В `drawer-nav` для **репетитора** (внутри `{% if user.profile.role == 'tutor' %}`): после пункта «Финансы» добавить пункт:
  - Ссылка: `{% url 'payment_receipts' %}`
  - Текст: «Заявки на пополнение»
  - Иконка: `<i class="fas fa-receipt"></i>`
  - Класс: `nav-item`, активность: `{% if request.resolver_match.url_name == 'payment_receipts' %}active{% endif %}`
- В `drawer-nav` для **ученика** (внутри `{% else %}`): после «Мои репетиторы» добавить пункт:
  - Ссылка: `{% url 'payment_receipts' %}`
  - Текст: «Оплаты»
  - Иконка: `fa-receipt`
  - Тот же паттерн `nav-item` и `active` по `payment_receipts`

### 2. Убрать legacy — core/templates/mobile/student_card.html
- Удалить кнопку, открывающую модалку ручного пополнения (например «Пополнить баланс» с `onclick="openModal('balance-modal')"`).
- Удалить целиком модалку `#balance-modal`, внутри которой форма с `name="update_balance"`, полями amount, date, description и кнопкой «Зачислить».
- На место кнопки вставить ссылку на раздел заявок: текст «Заявки на пополнение», href=`{% url 'payment_receipts' %}`, оформить в том же стиле, что и остальные кнопки в `s-actions-list` (например как `s-action-btn` по виду, но тег `<a>`).
- Под блоком с балансом (b-cards-row) добавить короткую подсказку: «Пополнение баланса — по заявкам ученика с чеками в разделе «Заявки на пополнение»» (можно со ссылкой на `payment_receipts`). Стиль: мелкий шрифт, цвет `var(--tg-hint)`.

### 3. Просмотр чека в модалке — core/templates/mobile/payment_receipts_tutor.html
- В каждой карточке заявки в блоке «Скан»: заменить простую ссылку на кнопку (или ссылку-кнопку), при клике открывающую модалку просмотра. Передавать в модалку URL файла и имя файла (например через `data-file-url="{{ r.file.url }}"` и `data-file-name="{{ r.file.name|default:'' }}"`).
- Добавить одну модалку (например `id="receiptViewModal"`), оформленную в стиле `.tg-modal`:
  - Заголовок «Чек» и кнопка закрытия.
  - Контент: для изображений (по расширению не .pdf) — тег `<img>` с `src`, установленным через JS, `max-width: 100%`, ограничение по высоте (например `max-height: 70vh`). Для PDF — блок с текстом «Файл в формате PDF» и ссылкой «Открыть PDF» с `href` из data и `target="_blank"`.
- JS: при клике по кнопке «Скан»/«Просмотр» читать `data-file-url` и `data-file-name`; если в имени есть `.pdf`, показывать блок PDF и скрывать img; иначе показывать img с этим url. Открывать модалку (`display: flex`). Закрытие: по клику на overlay и по кнопке «Закрыть»; при закрытии сбрасывать `img.src`.

## Ограничения
- Не добавлять новые view/url в Django. Не менять логику submit_receipt, approve_receipt, reject_receipt.
- Формы ученика и репетитора на мобильных страницах уже отправляют данные на те же эндпоинты; не менять `action` и имена полей форм.

## Результат
- Репетитор и ученик могут зайти в раздел чеков из мобильного меню.
- На карточке ученика нет ручного пополнения; есть переход в «Заявки на пополнение» и подсказка.
- Репетитор на странице заявок может открыть скан чека в модалке (фото — картинка, PDF — ссылка).
