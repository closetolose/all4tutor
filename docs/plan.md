# План: Глобальное обновление All4Tutors

Детальный пошаговый план разработки эпика «Глобальное обновление All4Tutors». План готов для передачи Implementers и Verifier.

---

## 1. Цели и правила

### Цели
- Улучшить навигацию и UI/UX (интеграция карт, модалки, сортировка).
- Внедрить визуальную индикацию и тегирование материалов.
- Усилить управление участниками и группами (бейджи групп, заявки на открепление).
- Добавить аналитику ДЗ/проверочных и уведомления о просрочках.
- Завершить и усилить финансовый учёт (чеки, подтверждение, баланс).
- Выделить админ-инструменты (заявки на открепление, управление пользователями).

### Технические ограничения
- Стек: Django (ORM, Templates), Vanilla JavaScript, CSS с переменными для тем. Docker (Nginx + Gunicorn).
- Без тяжёлых фронтенд-фреймворков (React/Vue). UI в рамках текущей дизайн-системы.
- Финансы: все операции, меняющие баланс или создающие транзакции, — только внутри `transaction.atomic()`.

### Глобальное правило аудита (обязательно)
**Перед разработкой любого пункта первым шагом всегда:** «Анализ существующего кода на наличие этой функции».  
Если функция уже есть — не писать с нуля, а восстановить работоспособность и улучшить UI/UX (верстка, отклик, модалки вместо алертов).
