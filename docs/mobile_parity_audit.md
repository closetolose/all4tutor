# Аудит Feature Parity: десктоп (1440px) vs мобильный (375px)

**Главное правило:** Каждое действие, доступное на десктопе, должно быть доступно и удобно на мобильном. Скрытие элементов через `display: none` или `.hidden-mobile` без мобильной альтернативы **запрещено**.

**Источники:** `core/templates/core/`, `core/templates/mobile/`, `core/views.py`, `tutor_project/urls.py`.

## План задач (Tasks) для Implementers

1. **Дашборд — Экспорт данных (репетитор)**  
   Добавить на мобильную главную (mobile/index.html) кнопку «Экспорт данных» (например, в шапку или в меню/overflow), открывающую модалку умного экспорта (те же поля, что в core: период, предмет, ученик, оплата, формат) с отправкой GET на export_lessons_csv.

2. **Подтверждения — Принять/Отклонить**  
   В mobile/confirmations.html заменить ссылки `<a href="...accept_request">` и `<a href="...reject_request">` на формы с method="post" и {% csrf_token %}, кнопки «Принять» и «Отклонить» как submit. Views принимают только POST.

3. **Мои файлы — Поиск**  
   Добавить на мобильную страницу файлов (mobile/files_library.html) поле поиска (форма GET с name="q") и кнопку сброса поиска при активном q.

4. **Мои файлы — Фильтр по тегам**  
   Добавить блок фильтра по тегам на мобильной странице файлов (чекбоксы или компактные чипы) и передавать tag в GET; в модалке загрузки файла добавить выбор тегов и опцию «новый тег».

5. **Мои файлы — Сортировка**  
   Добавить на мобильную страницу выпадающий список сортировки (по названию А→Я/Я→А, по дате новые/старые первые) с перезагрузкой страницы с параметром sort или JS-сортировкой списка.

6. **Мои файлы — Редактирование: теги**  
   В мобильной модалке редактирования файла (editFileModal) добавить поле/чекбоксы тегов и сохранение тегов при submit.

7. **Мои файлы — Подгрузка**  
   При наличии пагинации/load_more_files на десктопе реализовать на мобилке подгрузку следующих файлов (кнопка «Загрузить ещё» или infinite scroll) с тем же endpoint.

8. **Финансы — Сброс фильтра**  
   На mobile/finances.html добавить кнопку/ссылку «Сбросить» (href="{% url 'finances' %}") рядом с кнопкой «Показать».
