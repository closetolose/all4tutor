# ТЗ: UI/UX аудит и рефакторинг интерфейса All4Tutors

## 1. Цели и ограничения

- **Цель:** Устранить UI-баги и улучшить UX при сохранении текущей структуры экранов и палитры.
- **Ограничения:** Без тяжёлых UI-библиотек; только текущие CSS-классы и чистый CSS/JS; строго использовать переменные `--tg-bg`, `--tg-active`, `--tg-text`, `--tg-hint`, `--tg-separator`, `--tg-card-bg`, `--tg-red`, `--tg-green` и т.д. (десктоп — `static/css/style.css` и переменные `--primary`, `--border` и т.д. в рамках той же визуальной палитры). Полный редизайн запрещён.

## 3. Решения (Было → Станет)

### A1 — Единый вид карточки «Следующий урок»

- **Было:** Два определения `.next-lesson-card`: в base (градиент, белый текст) и в style.css (полупрозрачный фон).
- **Станет:** Оставить одно определение. Рекомендуется оставить вариант в `static/css/style.css` и удалить inline-блок из base; при необходимости скорректировать стили в style.css под градиент/цвета через переменные (например `--primary`), чтобы карточка в сайдбаре выглядела едино и не конфликтовала с палитрой.

### A2 — Модальное окно «Библиотека материалов»

- **Было:** Все стили `#fileExplorerModal` и содержимого заданы inline, жёсткие цвета.
- **Станет:** Перенести стили в `static/css/style.css` (или в один блок `<style>` в base с переменными). Использовать `var(--bg-white)`, `var(--border)`, `var(--text-muted)`, `var(--primary)`, `var(--primary-light)` и т.п., чтобы окно соответствовало теме и не дублировало палитру.

### A3 — Палитра блока фильтров дашборда

- **Было:** `.modern-filters` с градиентом `#1e3a8a` → `#3b82f6` и жёсткими цветами текста/фона.
- **Станет:** Заменить на градиент от `var(--primary)` к более тёмному оттенку или оставить синий акцент через переменные (например, добавить в :root `--filter-bg-start`, `--filter-bg-end` на базе `--primary`). Текст и поля — через `var(--tg-text)`, `var(--tg-hint)` или эквиваленты из style.css, чтобы сохранить читаемость и палитру.

### A4 — Дублирование и мусор в стилях карточек статистики

- **Было:** Несколько объявлений `.stat-card` и `.stat-card-homework` в index.html, мусорный комментарий в CSS.
- **Станет:** Собрать один блок правил для `.stat-card` и один для `.stat-card-homework` в index (или вынести в style.css). Удалить комментарий «REMOVED STRAY TEXT…». Цвета карточек (синий, зелёный, фиолетовый) оставить визуально теми же, но по возможности задать через переменные (например, оттенки от `--primary`, `--success` и т.д.).

### A5 — Единообразие мобильной палитры

- **Было:** В style.css при max-width 768px переопределяются `--tg-*` на тёмную тему; мобильный шаблон использует свой :root в mobile/base.
- **Станет:** Либо не переопределять глобальные `--tg-*` в style.css для 768px (если мобильная версия всегда рендерится отдельным шаблоном), либо согласовать один источник правды для мобильной темы (один :root или один media query), чтобы не было конфликта «десктоп на узком экране» vs «мобильный шаблон».

### A6 — Хлебные крошки

- **Было:** Разные классы и стили в base и в style.css (breadcrumb-wrapper, breadcrumb-item, breadcrumbs, breadcrumb-container).
- **Станет:** Унифицировать: один набор классов (например, `.breadcrumb-wrapper` + `.breadcrumbs` + `.breadcrumb-item` как в base) и все стили крошек держать в одном месте (style.css), убрать дубли из base. Использовать переменные для цветов ссылок и разделителей (`--tg-active`, `--tg-hint`).

### A7 — Адаптив панели массовых действий

- **Было:** `.bulk-bar` с `min-width: 450px` — риск переполнения на малых экранах.
- **Станет:** В media query (например, max-width: 900px или 768px) задать `min-width: auto`, `width: calc(100% - 30px)` или аналог, сохраняя отступы. Содержимое переносить при необходимости (flex-wrap, уменьшение отступов).

### A8 — Таблица расписания и длинный текст

- **Было:** Фиксированные проценты колонок, при переполнении текст может «ломать» вёрстку.
- **Станет:** Для ячеек с названием предмета и «С кем» задать `min-width: 0`, `overflow: hidden`, `text-overflow: ellipsis`, `white-space: nowrap` (или многострочный вариант с ограничением строк), чтобы длинные имена не ломали сетку.

### A9 — Названия файлов в библиотеке

- **Было:** `height: 40px; overflow: hidden` без ellipsis.
- **Станет:** Добавить `text-overflow: ellipsis`, при необходимости `white-space: nowrap` или ограничение по строкам с `line-clamp`, чтобы длинные названия обрезались с «…».

### A10 — Модальное окно файлов на малых экранах

- **Было:** Фиксированные max-height (85vh окно, 50vh сетка), риск обрезки контента.
- **Станет:** Для viewport до 900px (или 768px) уменьшить max-height сетки (например, 40vh или calc(100vh - 280px)), убедиться, что кнопка «Прикрепить» остаётся в зоне видимости (flex-контейнер с min-height для кнопки или sticky footer внутри модалки).

### A11 — Высота списка в мобильном file picker

- **Было:** `.mobile-fp-list` max-height 45vh; при открытой клавиатуре область списка сжимается.
- **Станет:** Использовать `max-height: min(45vh, calc(100vh - 320px))` или при открытии поля поиска уменьшать max-height списка через JS/класс, чтобы оставалось место под клавиатуру и кнопку «Готово».

### A12 — Класс .sidebar-link

- **Было:** В style.css есть `.sidebar-link.active` и hover, в base — только `.nav-item`.
- **Станет:** Либо переименовать в шаблоне класс в `.sidebar-link` и использовать общие стили, либо удалить неиспользуемые правила `.sidebar-link` из style.css, чтобы не было мертвого кода.

### B1–B5 — Touch targets не менее 44×44 px

- **Было:** Кнопки закрытия (32px, 36px), кнопки в selection-toolbar (36px), `.att-dot` (26px), сегменты с маленьким padding.
- **Станет:** Увеличить минимальные размеры до 44×44 px для интерактивных элементов (кнопки закрытия, кнопки в toolbar, сегменты — за счёт padding/min-height). Для `.att-dot` — увеличить до 44×44 px или оставить визуальный размер меньше, но расширить зону нажатия через padding/псевдоэлемент (hit area 44px, иконка по центру).

### B6 — Единый стиль полей в модалке файлов

- **Было:** Inline-стили с #ddd и белым фоном.
- **Станет:** Использовать классы вроде `.filter-select` / общие поля из style.css и переменные (`var(--border)`, `var(--bg-white)`), как в формах «Новое занятие» и фильтрах.

### B7 — Стили select в мобильном фильтре

- **Было:** Inline background для стрелки и цвет.
- **Станет:** Вынести в класс (например, в mobile base или общий блок стилей модалки), цвет стрелки — `var(--tg-active)`.

### B8 — Индикация загрузки

- **Было:** Только opacity и мелкий спиннер.
- **Станет:** Для кнопок при отправке — показывать спиннер внутри кнопки и состояние disabled; для карточек/таблицы — оставить лёгкое затемнение, при желании добавить мелкий скелетон или текст «Загрузка…» рядом с областью контента.

### B9 — Кнопки-иконки в таблице (десктоп)

- **Было:** Маленькая кликабельная область.
- **Станет:** Задать кнопкам минимум 44×44 px (или padding, увеличивающий hit area до 44px), сохраняя визуальный размер иконки.

### B10 — Единый вид полей форм

- **Было:** Часть полей с нативным видом браузера.
- **Станет:** Для input, select, textarea использовать общие классы (уже есть в проекте), единые `border-radius`, `border`, `padding`, `appearance: none` где нужно, с фокусом через `var(--primary)`.

### C1–C4 — Дубли и два набора переменных

- **Станет:**  
  - C1: как в A1 — один источник стилей для `.next-lesson-card`.  
  - C2: как в A4 — один блок для `.stat-card` и `.stat-card-homework`.  
  - C3: стили file explorer вынести в style.css или один блок в base, убрать дубли из index.  
  - C4: в документации зафиксировать: десктоп — переменные из style.css; мобильный base — `--tg-*`. При добавлении новых цветов использовать существующие переменные и не вводить новые хексы в inline. При необходимости в style.css добавить алиасы вида `--primary: var(--tg-active);` для мобильного, если когда-то шаблоны объединят.

## 4. Критерии приёмки

1. **Палитра:** Ни один новый или изменённый стиль не вводит жёстко заданные цвета для фона, текста, акцента и границ — только переменные (`--tg-*`, `--primary`, `--border`, `--text-main` и т.д.) в зависимости от того, какой базовый файл используется (core vs mobile).
2. **Структура:** Расположение блоков (сайдбар, дашборд, фильтры, таблица, карточки, модалки снизу на мобилке, по центру на десктопе) не меняется; меняется только вёрстка (сетка, overflow, отступы) и стили.
3. **Адаптивность:** Чёткое разделение: стили для `@media (max-width: 900px)` (и при необходимости 768px) не ломают мобильный шаблон и не конфликтуют с палитрой mobile/base.
4. **Touch targets:** Все интерактивные элементы на мобильных экранах (кнопки, ссылки-кнопки, чекбоксы, сегменты, точки посещаемости) имеют минимальную область нажатия 44×44 px (визуально или за счёт расширенной hit area).
5. **Формы и фильтры:** Поля input, select, textarea выглядят едино (скругление, границы, фокус), без «деревянного» вида нативных контролов где это возможно; используются общие классы и переменные.
6. **Обратная связь:** Состояния «Выделено», «Загрузка», «Ошибка» видны пользователю (подсветка, спиннер, сообщения); изменения не ломают Django-шаблоны (`{% for %}`, `{% if %}`) и существующий JS (Fetch, пагинация, открытие/закрытие модалок).
7. **Дублирование CSS:** Нет двух разных определений одного и того же компонента (next-lesson-card, stat-card, file explorer); мусорные комментарии в стилях удалены.
8. **Переполнение:** Длинные названия (ученики, предметы, файлы) не ломают сетку; везде, где нужно, используется `text-overflow: ellipsis` и при необходимости ограничение по высоте/строкам.

# ТЗ: Комплексное обновление функционала (UI, интеграции, оплаты)

Ниже — пакет из 8 улучшений в рамках текущей архитектуры, дизайн-системы (CSS-переменные) и стека (Vanilla JS, без тяжёлых фреймворков).

## Блок A: Финансы и оплаты

### UC-01: Отправка чека учеником

| Элемент | Требование |
|---------|------------|
| Форма | Ученик загружает чек: **сумма** (обязательно, > 0), **дата платежа** (обязательно, не в будущем), **файл** (скан/фото чека — изображение или PDF), опционально комментарий. |
| Доступ | Только для учеников; только в адрес репетитора, с которым есть подтверждённая связь (ConnectionRequest status=confirmed). |
| Валидация | Лимит размера файла (например 10 МБ), допустимые типы (image/*, application/pdf). |
| Критерий приёмки | После отправки создаётся запись PaymentReceipt (student, tutor, amount, receipt_date, file, status=pending). Репетитор видит заявку в разделе проверки оплат. |

### UC-02: Верификация репетитором

| Элемент | Требование |
|---------|------------|
| Уведомления/список | Репетитор видит заявки со статусом pending (список/страница или блок в интерфейсе). При желании — интеграция с существующим каналом уведомлений (например Telegram). |
| Просмотр | Репетитор видит: ученик, сумма, дата, скан (превью или ссылка на скачивание). |
| Действия | Кнопки «Подтвердить» и «Отклонить». |
| Подтверждение | В одной транзакции БД (transaction.atomic): статус чека → approved; создаётся Transaction (type=deposit); StudentBalance по паре (tutor, student) увеличивается на сумму чека. |
| Отклонение | Статус чека → rejected; баланс не меняется. |
| Критерий приёмки | Все операции атомарны; при подтверждении баланс и история транзакций корректно обновляются; при отклонении данные не меняются. |
