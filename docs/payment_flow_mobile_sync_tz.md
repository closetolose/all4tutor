# ТЗ: Синхронизация флоу оплаты Desktop → Mobile

## Анализ (выполнено)

### Бэкенд (Django)
- **URLs:** `payment_receipts`, `submit_receipt`, `approve_receipt`, `reject_receipt` — общие для десктопа и мобильной версии.
- **submit_receipt:** POST `tutor_id`, `amount`, `receipt_date`, `file`, `comment` (optional). Редирект на `payment_receipts` + messages.
- **approve_receipt / reject_receipt:** POST с `receipt_id`, CSRF. Редирект на `payment_receipts`.

### Десктоп
- Ученик: `core/payment_receipts_student.html` — кнопка «Отправить чек», модалка с формой, список «Мои чеки».
- Репетитор: `core/payment_receipts_tutor.html` — таблица заявок, кнопки Подтвердить/Отклонить, ссылка на скан (target="_blank").

### Мобильная версия (уже есть)
- Ученик: `mobile/payment_receipts_student.html` — форма в модалке (те же поля), список карточек. **Эндпоинт тот же** `submit_receipt`.
- Репетитор: `mobile/payment_receipts_tutor.html` — карточки заявок, формы Подтвердить/Отклонить. **Эндпоинты те же.** Скан — только ссылка (открывается в новой вкладке).

### Проблемы
1. В мобильном меню (drawer) **нет пункта «Оплаты» / «Заявки на пополнение»** — пользователь не может перейти на страницу чеков.
2. На мобильной карточке ученика (`mobile/student_card.html`) у репетитора есть кнопка **«Пополнить баланс»** и модалка с ручным вводом суммы/даты/комментария (POST `update_balance`) — legacy, нужно убрать и заменить на переход к флоу по чекам.
3. Просмотр чека на мобильном у репетитора только по ссылке (внешняя вкладка); желательно добавить просмотр изображения чека в модалке (для фото; PDF можно оставить по ссылке).

## Задачи для Implementer

### 1. Навигация (mobile/base.html)
- В блоке `drawer-nav` для **ученика** (`{% else %}` ветка): добавить пункт меню после «Мои репетиторы» (или после «Аналитика»):
  - Текст: «Оплаты» (или «Чеки на пополнение»).
  - Ссылка: `{% url 'payment_receipts' %}`.
  - Иконка: `fa-receipt`.
  - Класс `nav-item` и условие `active` по `request.resolver_match.url_name == 'payment_receipts'`.
- В блоке `drawer-nav` для **репетитора** (`{% if user.profile.role == 'tutor' %}`): добавить пункт меню (например после «Финансы»):
  - Текст: «Заявки на пополнение».
  - Ссылка: `{% url 'payment_receipts' %}`.
  - Иконка: `fa-receipt`.
  - Аналогично `active` по `payment_receipts`.

### 2. Убрать legacy «Пополнить баланс» (mobile/student_card.html)
- Удалить кнопку «Пополнить баланс» (или заменить на одну кнопку-ссылку).
- Удалить модалку `#balance-modal` с формой (поля amount, date, description, `name="update_balance"`).
- На место кнопки поставить ссылку/кнопку: «Заявки на пополнение» → `{% url 'payment_receipts' %}`, с короткой подсказкой, что пополнение только через чеки. Либо просто убрать кнопку и добавить под блоком баланса текст: «Пополнение баланса — по заявкам ученика в разделе «Заявки на пополнение»» со ссылкой на `payment_receipts`.

### 3. Просмотр чека в модалке (mobile/payment_receipts_tutor.html)
- В карточке заявки вместо одной ссылки «Скан» сделать: кнопка/ссылка «Скан» открывает модальное окно с просмотром чека.
- Если файл — изображение (jpg, png, webp и т.д.): показывать в модалке `<img src="{{ r.file.url }}" alt="Чек" style="max-width:100%; height:auto;">` и кнопку «Закрыть».
- Если PDF: в модалке можно показать ссылку «Открыть PDF» (target="_blank") или встроить через `<iframe>` (по желанию).
- Модалка: паттерн `.tg-modal`, закрытие по клику на overlay и по кнопке. Одна модалка на странице с динамической подстановкой URL чека (например через data-атрибут и JS) или отдельная модалка на каждую карточку с своим `r.file.url`.

## Чек-лист для Verifier
1. Ученик с мобильного: в меню есть «Оплаты», переход на страницу чеков, форма отправки с полями tutor_id, amount, receipt_date, file, comment отправляется на `submit_receipt`, после отправки редирект и сообщение.
2. Репетитор с мобильного: в меню есть «Заявки на пополнение», список заявок, кнопки Подтвердить/Отклонить работают (тот же бэкенд), после подтверждения баланс ученика обновляется.
3. На карточке ученика (mobile) у репетитора нет формы ручного пополнения баланса; есть ссылка/информация про «Заявки на пополнение».
4. Просмотр чека: репетитор может открыть скан в модалке (изображение) или по ссылке (PDF).
