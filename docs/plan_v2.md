# План v2: Глобальный рефакторинг интерфейса All4Tutors, тёмная тема и модуль аналитики

Детальный технический план эпика. Блок A обязателен первым (CSS-переменные и тема — основа для остальных блоков).

---

## 1. Подтверждение ограничений и последовательности

### Стек (строго)
- **Backend:** Django (ORM, шаблоны), Docker (Nginx + Gunicorn).
- **Frontend:** Vanilla JavaScript + CSS с переменными. Без React, Vue, Angular, Webpack, Vite.
- **Сторонний фронтенд:** только CDN (например Chart.js). Без сложных NPM-пакетов для фронта.
- **CSS:** все новые цвета — только через CSS-переменные (например `var(--bg-primary)`). Поддержка светлой/тёмной темы через `:root` и `[data-theme="dark"]`.
- **Параллель:** весь новый UI должен работать на десктопе и мобильной (проверка на 375px и 1440px).

### Порядок блоков
- **Блок A** (тёмная тема + дизайн/навигация) реализуется **первым**, чтобы переменные и тема были основой для блоков B, C, D.
- Зависимости: модалки (B), таблицы и кнопки (B) используют базовые стили и переменные из A; аналитика (D) использует те же переменные для графиков в тёмной теме.

## 2. Обзор архитектуры

```
┌─────────────────────────────────────────────────────────────────┐
│  Block A: CSS vars, [data-theme="dark"], toggle, cookie banner   │
│  → base.html, style.css, register.html, index/lessons (tooltip)  │
└───────────────────────────────┬─────────────────────────────────┘
                                │
        ┌───────────────────────┼───────────────────────┐
        ▼                       ▼                       ▼
┌───────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Block B      │     │  Block C        │     │  Block D        │
│  Finances UI  │     │  Help/FAQ       │     │  Analytics      │
│  Tutor card   │     │  Accordion      │     │  Models → Views  │
│  (modal, flex)│     │  placeholder    │     │  → Chart.js CDN  │
└───────────────┘     └─────────────────┘     └─────────────────┘
```

- **Данные:** существующие модели (`PaymentReceipt`, `ConnectionRequest`, `StudentPerformance`); новая модель для аналитики (см. блок D).
- **Фронт:** один глобальный `style.css` с переменными; скрипты в шаблонах или в `base.html`; Chart.js подключается по CDN только на страницах с графиками (finances, новая страница «Результаты»).
