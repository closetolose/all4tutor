# Результат этапа анализа: комплексное обновление All4Tutors

Документ содержит описание решений в формате «Было → Станет» по каждому UC и сводные критерии приёмки. Стек: Django, HTML/CSS, Vanilla JS; без смены глобальной структуры проекта.

---

## 1. Описание решений по UC

### Блок A: Финансы и оплаты

#### UC-01: Отправка чека учеником

| Аспект | Было | Станет |
|--------|------|--------|
| Действие | Ученик пополняет баланс только через ручное пополнение репетитором (карточка ученика). | Ученик может загрузить чек об оплате: сумма, дата платежа, скан/фото чека. |
| Форма | Нет формы загрузки чека. | Форма с обязательными полями: сумма (Decimal), дата платежа (date), файл (FileField, изображение/PDF). Опционально: комментарий. |
| Валидация | — | Сумма > 0; дата не в будущем; файл в пределах лимита размера (например 10 МБ); только свои репетиторы (связь confirmed). |
| Результат | — | Создаётся запись PaymentReceipt со статусом `pending`; репетитор видит заявку в разделе проверки оплат. |

#### UC-02: Верификация репетитором

| Аспект | Было | Станет |
|--------|------|--------|
| Уведомления | — | Репетитор видит список заявок на подтверждение (новые чеки); при желании — уведомление в интерфейсе или Telegram (по существующей логике). |
| Просмотр | — | Репетитор видит: ученик, сумма, дата, скан (превью/ссылка на скачивание). |
| Действия | — | Кнопки «Подтвердить» и «Отклонить». При подтверждении: в `transaction.atomic()` обновляется статус чека на `approved`, создаётся Transaction (deposit), увеличивается StudentBalance. При отклонении: статус `rejected`, баланс не меняется. |
| Логика баланса | Пополнение только ручное репетитором. | При подтверждении чека баланс ученика у этого репетитора увеличивается на сумму чека; создаётся запись в истории транзакций. |

## 2. Критерии приёмки (сводные)

1. **Финансы и оплаты**
   - Ученик может отправить чек (сумма, дата, файл) только в адрес репетитора, с которым есть подтверждённая связь.
   - Репетитор видит заявки в статусе «ожидает»; может открыть скан, подтвердить или отклонить.
   - При подтверждении в одной транзакции БД: статус чека → approved, создаётся Transaction (deposit), обновляется StudentBalance.
   - При отклонении статус чека → rejected, баланс не меняется.

2. **Модальное окно урока**
   - По клику открывается модалка с полными данными урока (предмет, время, длительность, формат, статус, локация, кнопка карт при адресе, заметки, материалы).
   - Нет конфликта с существующим tooltip (tooltip можно оставить для быстрого превью или убрать в пользу только модалки).

3. **Yandex Maps**
   - Кнопка «Открыть в Яндекс.Картах» показывается только если локация не начинается с `http://` или `https://`; ссылка: `https://yandex.ru/maps/?text=<urlencode(location)>`.

4. **Сортировка таблиц**
   - В основных таблицах (уроки на главной, файлы, при необходимости финансы/журнал) доступна сортировка по столбцам; направление и активный столбец отображаются.

5. **Цвета и hover**
   - При наведении на строку/карточку с цветовой индикацией пользовательский цвет не исчезает (hover использует тот же `--custom-entity-color` с усиленным фоном).
   - Репетитор может задать цвет для связи с учеником; в расписании репетитора отображается его цвет (по ученику/группе).

6. **Теги файлов**
   - Репетитор создаёт теги, привязывает их к файлам; в библиотеке работает фильтрация по тегам (совместно с поиском по названию).

7. **Карточка ученика**
   - Отображается блок с группами ученика (только группы текущего репетитора) со ссылками на карточки групп.

8. **Общее**
   - Все изменения не ломают существующие шаблоны (core/, mobile/), стили (CSS-переменные), цветовую индикацию (entity-color-indicator) и текущий JS (Vanilla).
   - Используются только существующие дизайн-переменные и структура проекта.
